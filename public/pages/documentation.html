API Documentation

This document outlines how to interact with the API, including available endpoints and expected request formats.

GET /customers

Retrieves a list of all customers.

GET /customers

POST /customers

Creates a new customer. Requires a JSON body with "name" and "email".

POST /customers
Content-Type: application/json

{
    "name": "Alice Smith",
    "email": "alice@example.com"
}

GET /products

Retrieves a list of all products.

GET /products

POST /products

Creates a new product. Requires a JSON body with "name" and "price".

POST /products
Content-Type: application/json

{
    "name": "Water Bottle",
    "price": 1.99
}

GET /vending-machines

Retrieves a list of all vending machines.

GET /vending-machines

POST /vending-machines

Creates a new vending machine. Requires a JSON body with "location".

POST /vending-machines
Content-Type: application/json

{
    "location": "Second Floor"
}

GET /transactions (Active Only)

Retrieves a list of all active (not marked as deleted) transactions.

GET /transactions

POST /transactions

Creates a new transaction. Requires a JSON body with "customerId", "productId", "vendingMachineId", "price", and "quantity".

POST /transactions
Content-Type: application/json

{
    "customerId": "uuid-goes-here",
    "productId": 1,
    "vendingMachineId": 1,
    "price": 1.50,
    "quantity": 1
}

PUT /transactions/:id

Marks a transaction as deleted based on its ID.

PUT /transactions/1
